Metodología BEM

La metodología BEM permite evitar los conflictos entre los distintos selectores. Nos vamos a basar en los nombres de las clases para ir asignando los estilos. Nos basaremos en el nombre de las clases para saber qué elementos estamos seleccionando. Veamos el siguiente código 

		
			<div class="form">
				<input type="text" class="name"/>
				<input type="password" class="pass"/>
			</div>
La idea para ir asignando los nombres de las clases es tomar la clase del contenedor (que en nuesto caso es el div) y anexar el valor de su atributo class con el valor del atributo class del objeto contenido de la siguiente manera class="form__name".
De esa manera se sabe que existe un contenedor de clase form que contiene a un elemento de clase "name".
Las clases quedarían de la siguiente manera:
		
			<div class="form">
				<input type="text" class="form__name"/>
				<input type="password" class="form__pass"/>
			</div>
Un escenario que puede darse es que deseemos cambiar sólo algunos atributos de un elemento del contenedor. Sea el siguiente código
		
			<div class="contact-form">
				<input type="text" class="contact-form__input"/>
				<input type="text" class="contact-form__input"/>
				<input type="password" class="pass"/>
			</;div>
Es posible que sólo deseemos aplicar estilo al primer input. Para ello hacemos contact-form__input:nth-child(posicionHijo)

Si se escribe en el primer input se notará que las letras se tornan rojas, mientras que las letras de los otros input siguen siendo negras. El nth-child nos permite seleccionar el n-ésimo hijo de un contenedor.
Otra forma de establecer, en el mismo ejemplo, que uno o más de los input tendrán un tratamiento diferenciado es usar una sintaxis similar a esta
<input type="text" class="contact-form__input--nombreEspecial"/>
Ese 'nombreEspecial' debe ser un identificador que permita aclarar que ese input (puede ser otro elemento html, el input es sólo para el ejemplo) tiene un tratamiento especial. Por ejemplo, podemos hacer <input type="text" class="contact-form__input--active"/>
Con eso aclaramos que ese input será el input activo en nuestro formulario. Luego, en css se hace .contact-form__input--active

Una cuestión particular a tener en cuenta es que los '__' se usan por bloque o secciones. Es decir, en los ejemplos anteriores la sección es el div con la clase "contact-form". Si algo es hijo de una sección, pero no es una sección o bloque en sí, la nomenclatura cambia. Sea el siguiente 

		
			<div class="contact-form">
				<p class="contact-form__p">
					<h2 class="contact-form__p-h2">

					</h2>
				</ p>
				
			<div/>
Vemos para la clase del h2 que no se ha hecho class="contact-form__p__h2 sino class="contact-form__p-h2.
Esto es así porque ese h2 interno a p no es un bloque, sino sólo un elemento interno a otro elemento que sí está en un bloque.
