¿Cómo agregar estilos CSS?

Una de las primeras formas de agregar estilos CSS a una etiqueta HTML es hacerlo de manera inline. Esto se refiere a usar el atributo style de la etiqueta de la siguiente manera:
<h1 style="modificacionDeAtributos">
Por ejemplo, para cambiar el color del texto de una etiqueta h1 se hace:
<h1 style="color:rgb(valorRojo,valorVerde,valorAzul)">
En este ejemplo, "color" es un atributo, una caracteristica de la etiqueta que podemos modificar para que tenga ciertas características.
Un problema que se tiene al usar estilos inline es que si deseamos aplicar el mismo color a varias etiquetas debemos aplicar style="restoDelCodigo" a cada una de ellas. Esto puede ser tedioso e impráctico.
Otra forma de aplicar estilos css al documento HTML es mediante el CSS interno. En la estructura de HTML una de las primeras etiquetas es la etiqueta <head>. Dentro de esta etiqueta debemos colocar la etiqueta <style></style>. Dentro de la etiqueta style debemos usar los llamados selectores.
Un selector servirá para indicar qué etiquetas html queremos reciban el estilo creado. Veamos el siguiente código 

            
                <head>
                    <style>
                    h1 {
                        color: red;
                        background-color:black;
                    }
                    </style>
                </head>

El código superior modificará el color de la letra y el color de fondo de todas las etiquetas h1 que haya en el documento en el cual se definió la etiqueta style. En este caso ek selector es esa palabra h1 que se coloca antes de las llaves.
Algo que debe resaltarse es que si se aplica un estilo inline y un estio interno a la misma etiqueta, el estilo que afectará a la etiqueta será el estilo inline.
La tercera forma de aplicar estilos css es mediante un archivo externo. Normalmente los arhivos externos css deberán ir dentro de una carpeta denominada css.
Para vincular un archivo css a un arhivo html debemos usar la etiqueta link dentro de la etiqueta head: 
            
                <head>
                <link href="rutaDelArchivoCss" rel="stylesheet"/>
                </head>

Ahora dentro del archivo css podemos usar el selector h1 y darle el estilo que se desee a la etiqueta h1.
Al igual que sucedia entre un estilo css inline y un estilo css interno, cuando un documento html tiene definido un estilo interno y está vinculado a un archivo CSS externo, el estilo que se aplique a un selector tendrá prioridad para el estilo interno. Entonces, las prioridades serán: inline->interno->archivo externo.
Por lo que he experimentado, si se tienen dos estilos sobre una etiqueta (por ejemplo un estilo interno y un archivo css externo) y ambos estilos están modificando el mismo atributo gana el que tenga más prioridad. Pero, si en ambos selectores se estan modificando atributos distintos, estos se combinan. Un ejemplo de lo expuesto puede verse en este documento donde en la etiqueta style se está modifica el color de letra y la alineación del texto, mientras que en el archivo estilos.css para el mismo selector se está modificando color de letra, posición del texto y color de fondo.
El resultado del título es una texto que tiene el color de letra y posición del texto indicados en la etiqueta style, pero su color de fondo está indicado en estilos.css.
También podemos forzar que un estilo en un archivo externo tome prioridad respecto de los estilos inline e interno. Eso se logram mediante la sentencia !important
            
                h1 {
                    color: green !important;
                    background-color:violet !important;
                }

Lsa última cuestiones importantes sobre prioridades es que la etiqueta style debe estar definida después de las etiquetas link que conectan el documento html con archivos css. Además, si los navegadores web tienen estilos propios estos tienen la prioridad más baja de todas.

Existe una forma de "llamar" una hoja de estilos dentro de otra hoja de estilos. La forma de hacerlo es mediante
@import url(rutaHojaEstilo)
A pesar de lo señalado se considera mala práctica usar @import porque al momento de cargar la página web se deben descargar ciertos recursos como ser fuentes, imagenes, etc. Cuando el navegador lee @import se detiene hasta leer todo el contenido y continúa cuando termina. Esto puede ralentizar la carga de la página. Con los preprocesarores como SASS, el @import si se puede usar, pero no en css puro.

Otro selector es el selector id. Los selectores id están ligados a un atributo de las etiquetas denominado id. El id de una etiqueta sirve para darle a esa etiqueta un identificador propio que los diferencie de cualquier otra etiqueta en el documento. Por lo tanto, dos etiquetas no pueden tener exactamente el mismo valor en su id.
Para colocar un id a una etiqueta se hace <h1 id="nombreId">
En el archivo css hacemos: 

            
                #nombreId {
                    color: green !important;
                    background-color:violet !important;
                }

La siguiente forma de establecer estilos es mediante clases. Las clases, a diferencia de los id, se pueden utilizar repetidas veces. La forma de declararlas en el documento css es haciendo .nombreClase{modificarAtributos}.
Luego se puede aplicar a un elemento html haciendo <h1 class="nombreClase">

Un elemento html puede tener varias clases. Sólo debemos colocar en el atributo class el nombre de las clases separadas por un espacio en blanco.
